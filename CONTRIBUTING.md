# Contributing to Recall

Thank you for taking the time to contribute!

## Getting started

1. **Fork** the repository
2. **Clone** your fork: `git clone https://github.com/your-username/recall.git`
3. **Install dependencies:** `npm install`
4. **Run in dev mode:** `npm run tauri dev`

## Workflow

- Create a branch for your change: `git checkout -b feat/my-feature`
- Make your changes
- Run type checking: `npm run check`
- Commit using [Conventional Commits](https://www.conventionalcommits.org/): `feat:`, `fix:`, `refactor:`, `docs:`, etc.
- Open a Pull Request against `master`

## Project structure

```
recall/
├── src/                        # SvelteKit frontend (TypeScript)
│   ├── lib/
│   │   ├── components/         # Svelte components
│   │   ├── api/                # Tauri command wrappers
│   │   ├── state.ts            # Global clipboard store
│   │   └── bindings.ts         # Auto-generated types from Rust (ts-rs)
│   └── routes/                 # SvelteKit pages
└── src-tauri/                  # Rust backend
    └── src/
        ├── clipboard.rs        # Clipboard watcher
        ├── history.rs          # In-memory + persistent history
        ├── commands.rs         # Tauri commands exposed to frontend
        ├── settings.rs         # Settings load/save
        └── models.rs           # Shared data types
```

## Guidelines

- Keep PRs focused — one feature or fix per PR
- Do not add dependencies without discussion
- The `bindings.ts` file is auto-generated by `ts-rs` at build time — do not edit it manually
- Prefer editing existing files over creating new ones

## Reporting bugs

Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md) and include:
- Your OS and version
- Steps to reproduce
- Expected vs actual behavior
